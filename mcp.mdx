<Info>
**What is MCP?**

The Model Context Protocol (MCP) is an open standard that enables AI agents to securely connect to external tools, data sources, and services. Think of it as a universal adapter that lets AI assistants interact with your business systems.
</Info>

## Overview

MCP provides a standardized way for AI agents (like Antigravity, Claude, or custom assistants) to:

- ðŸ“Š **Access data** - Read from databases, APIs, and files

## Creating Custom MCP Servers

You can create custom MCP servers for your specific business needs.

### Basic Server Structure

<CodeGroup>
```typescript server.ts
import { Server } from '@modelcontextprotocol/sdk/server/index.js';
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js';

// Create server instance
const server = new Server({
  name: 'custom-business-server',
  version: '1.0.0',
}, {
  capabilities: {
    tools: {},
  },
});

// Define a tool
server.setRequestHandler('tools/list', async () => {
  return {
    tools: [
      {
        name: 'create_lead',
        description: 'Create a new lead in the CRM',
        inputSchema: {
          type: 'object',
          properties: {
            name: { type: 'string' },
            email: { type: 'string' },
            phone: { type: 'string' },
          },
          required: ['name', 'email'],
        },
      },
    ],
  };
});

// Handle tool execution
server.setRequestHandler('tools/call', async (request) => {
  if (request.params.name === 'create_lead') {
    const { name, email, phone } = request.params.arguments;
    
    // Your business logic here
    const result = await createLeadInCRM({ name, email, phone });
    
    return {
      content: [
        {
          type: 'text',
          text: `Lead created successfully: ${result.id}`,
        },
      ],
    };
  }
  
  throw new Error(`Unknown tool: ${request.params.name}`);
});

// Start server
const transport = new StdioServerTransport();
await server.connect(transport);
```

```javascript package.json
{
  "name": "custom-business-mcp-server",
  "version": "1.0.0",
  "type": "module",
  "bin": {
    "custom-business-server": "./build/index.js"
  },
  "dependencies": {
    "@modelcontextprotocol/sdk": "^0.5.0"
  }
}
```
</CodeGroup>

## MCP Configuration

### Client Configuration

Most MCP clients use a JSON configuration file:

<CodeGroup>
```json ~/.config/mcp/config.json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_xxxxx"
      }
    },
    "custom-business": {
      "command": "node",
      "args": ["./path/to/custom-server.js"],
      "env": {
        "API_KEY": "your-api-key",
        "DATABASE_URL": "postgresql://..."
      }
    }
  }
}
```
</CodeGroup>

### Environment Variables

<Warning>
**Security Best Practice**

Never hardcode API keys or secrets in configuration files. Use environment variables:

```bash
export GITHUB_TOKEN="ghp_xxxxx"
export GHL_API_KEY="your-key"
export DATABASE_URL="postgresql://..."
```

Then reference them in your MCP config:
```json
{
  "env": {
    "GITHUB_PERSONAL_ACCESS_TOKEN": "${GITHUB_TOKEN}"
  }
}
```
</Warning>

### With Custom Agents

```typescript
import { Client } from '@modelcontextprotocol/sdk/client/index.js';

const client = new Client({
  name: 'my-agent',
  version: '1.0.0',
});

// Connect to MCP server
await client.connect(transport);

// List available tools
const tools = await client.request({
  method: 'tools/list',
});

// Call a tool
const result = await client.request({
  method: 'tools/call',
  params: {
    name: 'create_lead',
    arguments: {
      name: 'John Doe',
      email: 'john@example.com',
    },
  },
});
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Server Not Connecting">
    **Symptoms**: MCP server fails to start or connect
    
    **Solutions**:
    1. Check server is installed: `npm list -g @modelcontextprotocol/server-name`
    2. Verify configuration syntax: `cat config.json | jq`
    3. Check environment variables are set
    4. Review server logs for errors
    5. Ensure correct Node.js version (18+)
  </Accordion>

  <Accordion title="Authentication Errors">
    **Symptoms**: 401 Unauthorized or 403 Forbidden errors
    
    **Solutions**:
    1. Verify API tokens are valid and not expired
    2. Check token has required scopes/permissions
    3. Ensure environment variables are loaded correctly
    4. Test API credentials directly (e.g., with curl)
  </Accordion>

  <Accordion title="Tool Not Found">
    **Symptoms**: "Unknown tool" or tool not listed
    
    **Solutions**:
    1. Verify server is running: check process list
    2. List available tools to confirm registration
    3. Check tool name matches exactly (case-sensitive)
    4. Restart MCP client after server changes
  </Accordion>

  <Accordion title="Performance Issues">
    **Symptoms**: Slow responses or timeouts
    
    **Solutions**:
    1. Implement caching for frequently accessed data
    2. Add rate limiting to prevent overload
    3. Optimize database queries
    4. Use connection pooling
    5. Monitor server resource usage
  </Accordion>
</AccordionGroup>
